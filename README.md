# Zero Trust & Identity Lab Guide<br>
## Introduction
This is a lab guide designed for Junior Security Analysts focused on the transition from Traditional Perimeter Security to Zero Trust Architecture, based on NIST SP 800-207.
<br>

[Open the Full Lab Guide] (./docs/lab-guide.md)

## Pre-requisites
| Requirement | Details |
|------------| --------|
| OS | Ubuntu 20.04+ / Debian / Kali Linux |
| Identity | GitHub or Google Account (for SSO Login) |
| Skills Required | Basic Linux Terminal Usage |
| Time | ~2 hours |

## Learning Objectives
By the end of this lab, you will be able to:
- Explain the difference between perimeter security and Zero Trust
- Set up Identity-based network access using Tailscale
- Write ACL rules to enforce microsegmentation
- Configure Linux 'sudoers' to apply the Principle of Least Privilege
- Use an LLM (Claude / ChatGPT) to audit 'auth.log' security violations
